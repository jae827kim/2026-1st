# K-Living Solution 
> **Reward-based Knowledge Sharing System**

## 1. Business Purpose
* **통계적 배경**: 통계청 자료에 따르면 국내 1인 가구 수는 2024년 804만 가구로 전체의 36.1%를 차지하며 보편적인 주거 형태로 자리 잡았다.
* **필요성 증대**: 자취를 시작한 지 얼마 되지 않은 사회초년생과 고연차 사람들이 어울려 자취생 맞춤형 생활 솔루션을 공유할 필요성이 증대되었다.
* **기존 커뮤니티의 한계**: 명확한 보상 체계가 없어 답변의 신뢰도가 낮고 광고성 답변이 많아 정보의 선순환이 이루어지지 않는다.
* **프로젝트 목표**: 포인트 보상형 지식 공유 시스템을 구축하여 정보의 불균형을 해소하고, 고연차들이 포인트 획득 및 등급 상승을 목적으로 양질의 답변을 등록하게 유도한다.
* **데이터 재사용**: 채택된 답변은 아카이브에 저장되어 추후 다른 초보 자취생들에게 검색을 통해 제공되며 선순환을 일으킨다.

## 2. System Architecture & Concept 
본 프로젝트는 네트워크 기반의 실제 서버-클라이언트 구조를 Java 환경의 가상 서버 클래스와 파일 DB로 시뮬레이션한다.

| ID | Use Case | Purpose | Dynamics | Goals |
| :--- | :--- | :--- | :--- | :--- |
| **UC-01** | **Registration Info** | 사용자의 계정 정보와 자취 경력을 시스템에 등록함 | 사용자가 회원가입 양식을 제출할 때 | 새로운 사용자의 고유 식별자와 경력 데이터를 DB에 안전하게 생성함 |
| **UC-02** | **Login** | 등록된 사용자인지 확인하고 서비스 이용 권한을 부여함 | 사용자가 ID/PW를 입력하고 접속을 시도할 때 | 인증된 사용자에게 세션을 부여하고 개인화된 기능 접근을 허용함 |
| **UC-03** | **User Profile with Labeling** | 유저의 자취 숙련도를 표시하여 신뢰도를 확보함 | 로그인 직후나 타인이 해당 사용자의 정보를 조회할 때 | 사용자의 경력에 맞는 수식어(라벨)를 부여하여 답변의 공신력을 높임 |
| **UC-04** | **Post & Reward** | 질문 등록과 동시에 답변 보상을 위한 포인트를 예치함 | 사용자가 포인트를 설정하여 질문 게시를 확정할 때 | 포인트 차감 및 게시글 생성을 통해 지식 거래의 담보를 설정함 |
| **UC-05** | **Answer Selection** | 채택된 답변에 대해 보상 포인트를 확정하고 소유권을 이전함 | 질문자가 답변 중 하나를 '채택'했을 때 | 예치된 포인트가 답변자에게 정확히 전달되도록 정산 로직을 완결함 |
| **UC-06** | **Transaction Result** | 포인트 정산 성공 여부와 최종 상태를 유저에게 알림 | 포인트 정산 로직이 서버에서 종료된 직후 | 유저가 본인의 포인트 변동 내역과 처리 결과를 즉각적으로 확인하게 함 |
| **UC-07** | **Data Load** | DB의 데이터를 메모리로 불러와 로직 처리가 가능하게 함 | 프로그램 구동 시 혹은 데이터 동기화가 필요할 때 | 텍스트 파일에 저장된 정보를 객체화하여 시스템이 참조 가능한 상태로 만듦 |
| **UC-08** | **Data Save** | 생성된 데이터가 유실되지 않도록 DB에 기록함 | 신규 데이터 생성 및 작업이 종료될 때 | 메모리상의 변경 내용을 영구 저장 장치에 물리적으로 기록함 |
| **UC-09** | **Data Update** | 변동된 데이터 상태를 DB에 반영하여 최신성을 유지함 | 데이터값에 변화가 생겨 동기화가 필요할 때 | 포인트 잔액, 채택 여부 등 실시간으로 변하는 상태값을 정확히 갱신함 |
| **UC-10** | **System Statistics** | 관리자가 시스템 운영 상황과 포인트 흐름을 모니터링함 | 관리자가 운영자 페이지에서 통계를 요청할 때 | 포인트의 유통량과 유저 활동 지표를 분석하여 시스템의 안정성을 점검함 |
| **UC-11** | **Data Control** | 부적절한 사용자나 콘텐츠를 제어하여 건전성을 유지함 | 관리적 개입이나 유저 신고에 따른 처리가 필요할 때 | 신고된 게시글 삭제나 규정 위반 유저를 차단하여 커뮤니티 정체성을 관리함 |
| **UC-12** | **Answer Registration** | 사용자가 질문 게시글에 대해 자취 경험을 기반으로 답변을 등록함 | 사용자가 질문 상세 화면에서 답변 내용을 작성한 뒤 등록 버튼을 눌렀을 때 | 등록된 답변을 해당 질문과 연결하여 저장하고, 질문자가 채택할 수 있는 답변 목록에 추가함 |

## 3. Problem Statement
1. 실제 서버를 구축하지 않기 때문에 네트워크가 지원되지 않는 환경에서도 프로그램을 사용할 수 있다.
2. 백엔드 환경에서 로컬로 구동되기 때문에, 해당 프로그램이 설치된 데스크탑을 제외하고는 시스템을 사용할 수 없다.
3. 실제 DB 대신 파일 입출력 방식을 사용하여 정보를 저장하고 추출한다. 이 과정에서 복잡한 검색이나 데이터 필터링이 자유롭지 않기 때문에 간단하게 출력하도록 한다.
4. 로그인을 하지 않으면 프로그램의 핵심 기능인 질문 작성, 답변 채택, 포인트 거래 기능을 사용할 수 없다.
5. 모든 데이터가 User.txt와 Post.txt 파일에 의존하므로, 사용자가 임의로 파일을 수정하거나 삭제할 경우 데이터 무결성이 파괴될 수 있다.

## 4. Glossary
| 용어 | 설명 |
| :--- | :--- |
| **라벨링 / Labeling** | 회원가입 시 기입한 정보를 토대로 시스템이 유저의 경력 데이터를 판단하여 닉네임 옆에 수식어를 붙여주는 프로세스이다. |
| **포인트 / Point** | 질문 등록 시에 사용되는 비영리적 재화로, 여러 질문자가 여러 답변 중 최적의 답변을 고르면 답변 작성자에게 포인트가 전달된다. 포인트는 누적 포인트와 현재 포인트로 종류가 나뉘며 누적 포인트에 따라 등급이 부여되고, 현재 포인트를 사용해 질문을 등록할 수 있다. |
