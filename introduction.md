# 1. Introduction

## 1) Executive Summary

최근 국내 1인 가구 수는 지속적으로 증가하고 있으며, 통계청 자료에 따르면 2024년 국내 1인 가구는 804만 가구로 전체 가구의 36.1%를 차지한다. 이는 1인 가구가 더 이상 특수한 주거 형태가 아니라 현대 사회에서 보편적인 생활 방식으로 자리 잡았음을 보여준다.

하지만 자취를 처음 시작한 사회초년생이나 대학생들은 생활 과정에서 다양한 문제를 겪는다. 예를 들어 월세 계약, 공과금 관리, 식재료 보관, 생활용품 구매, 청소, 벌레 퇴치, 집안 관리 등 실생활과 밀접한 문제들은 경험이 부족한 초보 자취생에게 큰 어려움이 될 수 있다. 반면 오랜 기간 자취를 해온 고연차 자취생들은 이러한 문제를 해결할 수 있는 경험과 노하우를 가지고 있지만, 이를 체계적으로 공유할 수 있는 구조는 부족하다.

기존 커뮤니티에서도 자취 관련 질문과 답변은 이루어지고 있으나, 명확한 보상 체계가 없기 때문에 답변의 품질과 신뢰도를 보장하기 어렵다. 또한 광고성 답변이나 무성의한 답변이 포함될 수 있어 사용자가 실제로 도움이 되는 정보를 찾는 데 어려움을 겪는다. 이러한 문제는 정보의 선순환을 방해하고, 초보 자취생과 숙련된 자취생 사이의 정보 불균형을 심화시킨다.

이를 해결하기 위해 본 프로젝트에서는 **K-Living Solution**을 제안한다.  
K-Living Solution은 자취 생활에 필요한 지식과 경험을 공유하는 **Reward-based Knowledge Sharing System**이다. 사용자는 질문을 등록할 때 보상 포인트를 설정하고, 답변자들은 해당 보상을 얻기 위해 양질의 답변을 작성한다. 질문자는 여러 답변 중 가장 적절한 답변을 채택할 수 있으며, 채택된 답변자에게 포인트가 지급된다.

또한 채택된 답변은 아카이브 형태로 저장되어 추후 다른 초보 자취생들이 검색을 통해 다시 활용할 수 있다. 이를 통해 단순한 일회성 질의응답이 아니라, 자취 생활 지식이 축적되고 재사용되는 구조를 형성할 수 있다.

## 2) Business Goals

K-Living Solution의 목적은 초보 자취생과 고연차 자취생 사이의 정보 불균형을 줄이고, 실생활에 도움이 되는 자취 지식을 효율적으로 공유할 수 있도록 하는 것이다.

초보 자취생은 질문을 통해 자신이 겪고 있는 생활 문제에 대한 해결책을 얻을 수 있으며, 고연차 자취생은 자신의 경험을 바탕으로 답변을 제공하고 포인트를 획득할 수 있다. 획득한 포인트와 누적 포인트는 사용자의 등급 및 신뢰도를 나타내는 기준으로 활용된다.

또한 시스템은 회원가입 시 입력된 자취 경력 정보를 바탕으로 사용자에게 라벨을 부여한다. 이를 통해 답변자의 자취 숙련도를 다른 사용자들이 쉽게 확인할 수 있으며, 답변의 신뢰성을 판단하는 데 도움을 줄 수 있다.

K-Living Solution의 주요 Business Goals는 다음과 같다.

- 초보 자취생이 자취 생활 중 발생하는 문제에 대해 신뢰할 수 있는 답변을 얻을 수 있도록 한다.
- 고연차 자취생이 자신의 경험과 지식을 공유하도록 포인트 보상 체계를 제공한다.
- 질문자가 보상 포인트를 예치하고 답변을 채택하는 구조를 통해 답변의 품질을 높인다.
- 채택된 답변을 아카이브에 저장하여 이후 다른 사용자가 검색하고 재사용할 수 있도록 한다.
- 사용자 라벨링과 누적 포인트 등급을 통해 답변자의 신뢰도와 활동 수준을 확인할 수 있도록 한다.
- 관리자가 시스템 통계와 사용자 활동을 확인하여 포인트 흐름과 커뮤니티 운영 상태를 점검할 수 있도록 한다.
- 부적절한 게시글이나 규정 위반 사용자를 제어하여 건전한 자취 지식 공유 커뮤니티를 유지한다.

## 3) Technical Goals

K-Living Solution은 실제 서버와 데이터베이스를 구축하지 않고, Java 환경에서 서버-클라이언트 구조를 가상 서버 클래스와 파일 입출력 방식으로 시뮬레이션하는 시스템이다. 따라서 네트워크가 지원되지 않는 환경에서도 프로그램을 실행할 수 있으며, 로컬 데스크탑 환경에서 시스템의 핵심 기능을 구현할 수 있다.

시스템은 사용자 정보와 게시글 정보를 각각 `User.txt`, `Post.txt` 파일에 저장한다. 프로그램이 실행될 때 파일에 저장된 데이터를 메모리로 불러오고, 사용자의 회원가입, 질문 등록, 답변 채택, 포인트 정산 등으로 인해 데이터가 변경되면 파일에 다시 저장하거나 갱신한다.

사용자는 회원가입을 통해 계정 정보와 자취 경력을 등록하며, 시스템은 이를 기반으로 사용자에게 적절한 라벨을 부여한다. 로그인에 성공한 사용자만 질문 작성, 답변 채택, 포인트 거래와 같은 핵심 기능을 사용할 수 있도록 제한한다. 이를 통해 시스템 기능 접근 권한을 관리하고, 사용자별 데이터를 구분할 수 있도록 한다.

포인트 거래 기능에서는 질문자가 질문 등록 시 보상 포인트를 예치하고, 답변이 채택되면 해당 포인트가 답변자에게 이전된다. 이 과정에서 포인트 잔액, 누적 포인트, 채택 여부 등의 데이터가 정확하게 갱신되어야 한다. 또한 정산 결과는 사용자에게 즉시 출력되어 포인트 변동 내역과 처리 결과를 확인할 수 있도록 한다.

K-Living Solution의 주요 Technical Goals는 다음과 같다.

- Java를 기반으로 시스템의 주요 기능을 구현한다.
- 실제 서버 대신 가상 서버 클래스를 사용하여 서버-클라이언트 구조를 시뮬레이션한다.
- 실제 DB 대신 `User.txt`, `Post.txt` 파일을 사용하여 데이터를 저장하고 불러온다.
- 프로그램 실행 시 파일 데이터를 메모리로 로드하여 시스템 로직에서 사용할 수 있도록 한다.
- 회원가입, 로그인, 질문 등록, 답변 채택, 포인트 정산 과정에서 발생하는 데이터 변경 사항을 파일에 저장한다.
- 사용자 ID, 비밀번호, 자취 경력, 포인트, 등급, 게시글 정보 등을 파일 기반으로 관리한다.
- 로그인한 사용자만 핵심 기능을 사용할 수 있도록 접근 권한을 제한한다.
- 포인트 예치, 차감, 지급, 누적 포인트 갱신이 정확하게 이루어지도록 한다.
- 관리자가 시스템 통계와 사용자 활동 정보를 확인할 수 있도록 한다.
- 파일이 임의로 수정되거나 삭제될 경우 데이터 무결성이 훼손될 수 있으므로, 기본적인 데이터 검증 로직을 포함한다.

## 4) Goals

이번 Analysis 보고서에서는 K-Living Solution의 사용자와 관리자가 시스템에서 어떤 기능을 사용할 수 있는지 구체적으로 분석한다. 이를 위해 Actor와 Use Case의 관계를 Use Case Diagram으로 표현하고, 각 Use Case에 대한 Description을 작성하여 기능의 흐름과 예외 상황을 명확하게 정의한다.

또한 Domain Analysis를 통해 시스템 구현에 필요한 주요 클래스들을 도출하고, 각 클래스가 어떤 역할을 수행하는지 설명한다. 특히 사용자 정보 관리, 게시글 관리, 포인트 정산, 데이터 로드 및 저장, 관리자 통계 기능을 중심으로 시스템 구조를 분석한다.

마지막으로 User Interface Prototype에서는 사용자가 회원가입, 로그인, 질문 등록, 답변 채택, 포인트 정산 결과 확인 등의 기능을 어떻게 사용할 수 있는지 간단한 화면 구성을 통해 제시한다.

본 문서를 통해 K-Living Solution이 어떤 목적을 가지고 있으며, 사용자가 시스템과 어떤 방식으로 상호작용하는지, 그리고 이를 구현하기 위해 어떤 데이터와 클래스 구조가 필요한지 이해할 수 있을 것이다.
