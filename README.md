# K-Living Solution 
> **Reward-based Knowledge Sharing System for Single-Person Households**

통계청 자료에 따르면 국내 1인 가구는 2024년 기준 전체의 36.1%를 차지하며 보편적인 주거 형태로 자리 잡았음. 본 프로젝트는 자취 초보자의 수요와 고연차의 노하우를 **포인트 보상 체계**로 연결하여 정보 불균형을 해소하고 양질의 자취 정보를 선순환시키는 백엔드 기반 시스템임.

---

## 1. Business Purpose
* **정보의 선순환**: 광고성 답변이 아닌, 숙련된 자취생의 실전 노하우를 아카이브화하여 재사용함.
* **신뢰도 시스템**: 누적 획득 포인트 및 채택 횟수를 기반으로 사용자 등급을 부여하여 정보 제공자의 숙련도와 신뢰도를 표시함.
* **카테고리별 전문성**: 부동산, 생활, 청소, 요리 등 카테고리를 분류하여 해당 분야를 토론할 수 있는 공간을 제공함.

---

## 2. System Architecture & Concept
본 프로젝트는 네트워크 기반의 실제 서버-클라이언트 구조를 **Java 환경의 가상 서버 클래스와 파일 DB**로 시뮬레이션한다.

### **Key Concepts (Concept of Operation)**
| ID | Use Case | Purpose | Dynamics (Trigger) |
| :--- | :--- | :--- | :--- |
| **UC-01** | **Registration** | 새로운 사용자의 계정 정보와 자취 경력을 시스템에 등록함 | 새로운 사용자가 서비스 이용을 위해 회원가입 양식을 제출할 때 |
| **UC-03** | **Labeling** | 유저의 자취 숙련도를 표시하여 지식 공유의 신뢰도를 확보함 | 로그인을 성공한 직후나 타인이 해당 사용자의 정보를 조회할 때 |
| **UC-04** | **Post & Reward** | 질문 등록과 동시에 답변 보상을 위한 포인트를 예치함 | 사용자가 질문글을 작성하고 보상 포인트를 설정하여 게시를 확정할 때 |
| **UC-05** | **Selection** | 채택된 답변에 대해 보상 포인트를 확정하고 소유권을 이전함 | 질문자가 등록된 답변 중 가장 도움이 된 것을 '채택'했을 때 |
| **UC-07** | **Data Load** | DB의 데이터를 메모리로 불러와 로직 처리가 가능하게 함 | 프로그램 구동 시 혹은 최신 데이터 업데이트를 위해 새로고침할 때 |

##  3. Problem Statement (Technical Constraints)
본 프로젝트는 핵심 로직의 안정적인 구현을 위해 다음과 같은 제약 조건을 가짐.

1. **Local 구동 환경**: 실제 서버를 구축하지 않아 네트워크가 지원되지 않아도 사용 가능하지만, 로컬 데스크탑 환경으로 실행이 제한됨.
2. **File I/O 기반 DB**: 실제 DB 대신 `User.txt`와 `Post.txt` 파일을 사용하여 정보를 저장하고 추출함.
3. **가상 서버 클래스**: 네트워크 통신을 직접 구현하는 대신 `Server` 클래스가 요청을 중계하고 로직을 처리하는 가상의 서버 역할을 수행함.
4. **출력 방식의 간소화**: 복잡한 검색이나 필터링이 자유롭지 않아 통계나 포인트 현황 등은 텍스트 기반으로 간단하게 출력함.
5. **로그인 필수**: 로그인을 하지 않으면 질문 작성, 답변 채택, 포인트 거래 등 핵심 기능을 사용할 수 없음.

## 4. Glossary
| 용어 | 설명 |
| :--- | :--- |
| **포인트 / Point** | 질문 등록 시 사용되는 비영리적 재화로, 최적 답변 채택 시 작성자에게 전달됨. |
| **라벨링 / Labeling** | 가입 시 기입한 정보를 토대로 시스템이 유저의 경력 데이터를 판단하여 수식어를 붙여주는 프로세스임.
| **현재 포인트** | 질문 등록 등 기능 이용에 즉시 사용할 수 있는 가용 포인트임. |
| **누적 포인트** | 유저가 획득한 총 포인트 합계로, 사용자 등급 산정의 기준이 됨. |
